About This Project

Credit Card Approval Prediction System

A production-ready machine learning web application built as a B.Tech final year project, demonstrating end-to-end ML engineering from data to deployment.

Overview

What is CreditAI?

Banks process thousands of credit card applications daily. Manual review is slow, inconsistent, and expensive. CreditAI automates this process using a machine learning model trained on the Kaggle Credit Card Approval Dataset.

The system analyses 17 applicant features — financial, demographic, and employment — and predicts whether the application should be approved or rejected, along with a confidence percentage and risk category.

Machine Learning

Models Trained & Compared

📈
Logistic Regression

Baseline linear model. Fast to train, highly interpretable.

Strengths

Probability calibration · No overfitting · Fast inference

🌳
Decision Tree

Rule-based classifier. Fully interpretable decision paths.

Strengths

Human-readable · Handles non-linearity · No scaling needed

🌲
Random Forest BEST

Ensemble of trees. Best CV ROC-AUC on Kaggle data.

Strengths

Robust · Feature importance · Prevents leakage

⚡
XGBoost

Gradient boosted trees. High precision but lower CV ROC-AUC.

Strengths

Fast inference · Scalable · Regularisation

Data

Kaggle Credit Approval Dataset

The dataset is sourced from the Kaggle Credit Card Approval Prediction Challenge. It contains realistic applicant demographic and financial records.

  • 36,457 instances (after merging application and credit records)
  • 17 features — 8 numeric, 9 categorical
  • Binary target: Approved (+) or Rejected (−)
  • Class distribution: Highly imbalanced (few rejections)
  • Source: Realistic financial sector data
